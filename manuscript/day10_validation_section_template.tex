\subsection{Day-10 numerical convergence and reference validation}
To strengthen the reproducibility of the PCS evidence, a Day-10 validation run was added for the accepted single-band C and S operating points. The Day-10 run repeats the paired PCS-minus-uniform comparison over larger symbol counts and fixed SSFM resolution. The stability gate compares the two largest symbol counts and accepts a case only when the absolute GMI-gain drift is below the configured threshold and the PCS gain remains positive. This convergence check is used to avoid over-interpreting short waveform trials.

The Day-10 output also writes raw per-seed paired-gain tables and an optional external-reference comparison template. When a filled \texttt{validation\_data/gnpy\_day10\_reference.csv} file is supplied, the script reports the GSNR RMSE between the O-E-S-C-L waveform result and the external GNPy/GN/EGN reference. If this file is not supplied, the manuscript should describe the result only as an internal analytical GN-style sanity check and not as formal external GNPy validation.
